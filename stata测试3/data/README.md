# 工商注册数据清洗、合并与指标构造说明

## 1. 文件分工

当前这套处理流程主要由两部分组成：

- `清洗合并.do`
  负责对年度工商注册原始数据进行清洗，并将各年份处理后的结果保存为年度 `dta` 文件。

- `计算指标.do`
  负责在合并后的总样本上计算企业进入、退出及相关比率指标，并输出汇总结果表。

如果中间使用了年度纵向合并脚本，则该步骤的作用是：

- 将 `1948-2023` 年各年度处理后的 `dta` 文件纵向追加，形成总样本 `merged_1948_2023.dta`。

---

## 2. 原始数据到年度清洗样本的逻辑

`公司注册数据.do` 的核心是按年份循环处理 `1948-2023` 年的年度数据文件。

由于 `2022-2023` 年与更早年份在字段结构上略有差异，代码将其分成两段循环分别处理，但整体思路一致。

### 2.1 按年份循环读取原始数据

脚本逐年读取年度 `dta` 文件，对每一年的企业注册数据做相同口径的清洗。

### 2.2 样本筛选

年度清洗过程中，主要做了以下筛选：

- 删除关键字段为空的观测。
- 保留企业名称中包含“公司”的样本。
- 删除企业名称为空或明显无效的观测。

这一步的目标是把研究对象尽可能限定为正式**工商企业**主体，而不是其他非公司类记录或信息残缺记录。

### 2.3 构造进入退出变量 `enter`

代码中构造了一个核心变量 `enter`：

- `enter = 1` 表示企业进入
- `enter = -1` 表示企业退出

退出的识别方式是根据企业状态字段中的关键词进行判断。代码逻辑表明，凡是状态中出现与注**销、吊销、撤销**等退出含义相关的关键词时，就将 `enter` 从 `1` 改为 `-1`。

因此，`enter` 可以理解为企业状态变化在样本中的简化编码：

- 正常进入样本的企业记为 `1`
- 已经退出经营状态的企业记为 `-1`

### 2.4 保留核心字段

清洗代码只保留后续分析需要的核心变量，主要包括：

- 企业名称及标准化后的企业名称键值
- 行业信息
- 省份、城市
- 注册资本或相关金额变量
- 成立年份
- `enter`

这样做的目的是减少无关变量，降低后续合并和统计时的数据体积。

### 2.5 同一年内去重

年度清洗中包含两轮去重逻辑：

#### 第一轮：按原始名称或核心标识去重

代码先基于某个核心字符串变量分组，并计算每条记录的非缺失信息量：

- 对每个字段生成是否非缺失的标记
- 将这些标记加总，得到每条记录的信息完整度
- 在同组记录中保留信息最完整的一条

这一轮的目的是在同一主体有多条重复记录时，优先保留信息更完整的观测。

#### 第二轮：按标准化企业名称键 `biddername_key` 去重

代码对企业名称做了进一步标准化处理，包括：

- 替换部分异常或不统一的名称写法
- 去除括号说明、附加字样等噪声信息
- 提取以“公司”为结尾的主体名称
- 对名称做长度截断，生成统一的名称键值 `biddername_key`

然后再次按 `biddername_key` 分组，用“非缺失字段更多”的原则保留最完整记录。

这一轮是整个清洗中非常关键的一步，其作用是：

- 提高同一企业名称在不同写法下的可识别性
- 降低同一年内重复企业记录对统计结果的干扰

### 2.6 数值变量清洗

代码还对若干金额或资本变量进行了数值化处理，典型逻辑包括：

- 从字符串中提取数字部分
- 去掉逗号等格式符号
- `destring` 转成数值变量
- 对缺失字段进行合理填补

这一步的目的是让后续的统计分析可以直接使用金额或资本变量，而不是停留在字符形式。

### 2.7 生成年份变量并保存年度清洗结果

如果某些观测缺少年份，代码会用当前循环年份补齐。最后，每一年清洗后的样本都会保存为年度 `dta` 文件。

因此，年度清洗的输出结果可以理解为：

- 一个年份对应一个清洗后的年度样本
- 每个年度样本内部已经做了名称标准化、数值变量处理和同年去重

## 3. 跨年份合并逻辑

在年度清洗完成后，需要把所有年份的数据合并成一张总表，以便计算地区和行业层面的进入退出指标。

### 3.1 纵向追加

跨年份合并采用的是最标准的 Stata 纵向合并方式：

- 先读取 `1948.dta`
- 再按年份循环将 `1949-2023.dta` 依次 `append using`

这一步是“纵向合并”，不是“横向匹配”。也就是说：

- 每一年是一批新的观测
- 合并方式是把观测堆叠起来
- 最终形成一个长表

### 3.2 压缩与保存

合并完成后，代码使用 `compress` 压缩变量存储类型，以减小文件体积，随后保存为总样本，例如：

- `merged_1948_2023.dta`

这一总样本是后续指标计算的基础数据。

---

## 4. 指标构造逻辑

`calc_enter_exit_indicators.do` 的作用是基于合并后的总样本，计算地区和行业层面的企业进入退出指标。

当前代码中使用的核心变量为：

- `省份`
- `城市`
- `成立年份`
- `行业门类` 或 `行业`
- `enter`

### 4.1 基础变量

首先构造两个 0-1 标记变量：

- `entry_count = (enter == 1)`
- `exit_count = (enter == -1)`

含义是：

- 若该观测代表进入企业，则 `entry_count = 1`
- 若该观测代表退出企业，则 `exit_count = 1`

### 4.2 汇总口径

代码分别构造四类统计单元：

- 市级，地区-年份
- 省级，地区-年份
- 市级，地区-行业门类-年份
- 省级，地区-行业门类-年份

之后再把这四部分结果纵向追加成一张总表。

也就是说，最终指标表本身也是一个“长表”，只是这个长表里的每一行代表一个统计单元，而不是一家企业。

### 4.3 指标计算方法

对每一个统计单元，代码都计算以下变量：

- `enter_sum`
  含义：`enter` 的加总值，进入记 `+1`，退出记 `-1`

- `entry_count`
  含义：进入企业数

- `exit_count`
  含义：退出企业数

- `firm_total`
  含义：该统计单元中的样本企业总数

- `entry_rate = entry_count / firm_total`
  含义：进入率

- `exit_rate = exit_count / firm_total`
  含义：退出率

- `net_rate = enter_sum / firm_total`
  含义：净进入率

这里需要特别说明：当前进入率和退出率的分母采用的是“该统计单元当年的样本企业总数”，因此它反映的是一种样本占比口径。如果后续研究需要更标准的动态口径，也可以改为以上一年存量企业数为分母。

### 4.4 区分不同统计层级

为了把多个层级合并进一张总表，代码专门设置了两个分类变量：

- `统计口径`
  用于区分：
  - `地区-年份`
  - `地区-行业门类-年份`

- `地区层级`
  用于区分：
  - `省级`
  - `市级`

此外还设置：

- `地区名称`
  省级时等于省份名称
  市级时等于城市名称

因此，在后续分析时可以通过筛选这两个分类变量来获得任意一种统计结果，而不需要分别读取多张表。

## 5. 最终输出结果

根据当前代码，最终主要输出包括两类：

### 5.1 清洗并合并后的企业级总样本

- `merged_1948_2023.dta`

这是企业级长表，每一行对应一条企业观测。

### 5.2 进入退出指标总表

- `all_enter_exit_indicators.dta`
- `all_enter_exit_indicators.csv`

这是一张统计结果总表，已经把以下四类结果纵向合并在一起：

- 市级，地区-年份
- 省级，地区-年份
- 市级，地区-行业门类-年份
- 省级，地区-行业门类-年份

常用筛选方式如下：

#### 只看市级地区-年份

- `统计口径 == "地区-年份"`
- `地区层级 == "市级"`

#### 只看省级地区-年份

- `统计口径 == "地区-年份"`
- `地区层级 == "省级"`

#### 只看市级地区-行业门类-年份

- `统计口径 == "地区-行业门类-年份"`
- `地区层级 == "市级"`

#### 只看省级地区-行业门类-年份

- `统计口径 == "地区-行业门类-年份"`
- `地区层级 == "省级"`
