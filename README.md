# GPT5.4 Paper Automated Testing

本仓库用于整理和展示一个以 `VS Code + GPT-5.4 API + Stata MCP 插件 + LaTeX` 为核心的论文自动化写作与实证分析工作流。当前目录包含三个相对独立的 Stata 测试项目，每个项目都对应一篇论文或一组论文产出，用于验证“从数据处理/理论建模，到 Stata 分析，再到 LaTeX 成稿”的自动化链路是否顺畅。

整个流程的重点不是单纯生成文本，而是把研究过程拆成可以复用的几个模块：

1. 在 `VS Code` 中组织项目目录、脚本、论文模板和输出文件。
2. 通过 `GPT-5.4 API` 协助研究设计、脚本编写、结果解读与论文结构生成。
3. 通过 `Stata MCP` 插件把模型交互与 Stata 执行连接起来，让 `.do` 文件分析、表格导出和图形生成可以纳入统一自动化流程。
4. 以 `LaTeX` 作为论文成稿媒介，把 Stata 输出的表格片段、图形和统计摘要嵌入论文主文档。
5. 最终形成“代码 - 数据处理 - 结果产出 - 论文编译”一体化的可复现研究目录。

## 自动化实现流程

当前自动化链路可以概括为下面这条主线：

### 1. 研究问题生成与写作框架搭建

- 在 `VS Code` 中维护项目目录、脚本、文档和中间结果。
- 通过 `GPT-5.4 API` 辅助生成研究问题、变量设计、论文大纲、机制解释与章节写作框架。
- 明确每个测试项目是“理论型论文”还是“数据驱动型论文”，并决定后续是否需要 Stata 实证分析。

### 2. Stata 脚本自动化分析

- 使用 `Stata MCP` 插件调用 Stata，执行 `.do` 文件或逐段调试代码。
- 将分析拆分为 `master do-file -> 数据清洗/分析 do-file -> 输出结果` 的结构。
- 让 Stata 自动生成：
  - 中间数据集 `.dta`
  - 描述统计表、回归表等 `.tex` 片段
  - 论文插图 `.png`
  - 日志文件 `.log`

### 3. LaTeX 论文拼装

- 以 `LaTeX` 作为统一写作媒介。
- 在论文主文件中通过 `\input{...}` 引入 Stata 自动导出的表格片段。
- 直接引用 Stata 输出图形，使“模型结果更新 -> 图表更新 -> 论文更新”形成闭环。
- 最终编译得到 PDF 论文成稿。

### 4. 迭代式研究协同

- `GPT-5.4 API` 负责协助解释结果、润色论文、补充理论机制和重构章节逻辑。
- `Stata MCP` 负责把新的想法快速转成可执行分析。
- `VS Code` 负责统一管理脚本、论文和版本。
- `LaTeX` 负责承接最终的学术写作输出。

这意味着当前工作流本质上是一个“AI 协助研究 + Stata 执行分析 + LaTeX 组织成稿”的闭环，而不是只做文本生成。

## 三个 Stata 测试 / 三篇论文

### 1. `stata测试`

这是当前最完整的一篇实证论文测试，主题为：

**《组织规模、人力资本结构与上市公司违规风险：来自中国上市公司的面板证据》**

#### 研究内容

- 基于 2015-2022 年中国上市公司面板数据；
- 研究组织规模、技能型员工投入与企业违规风险之间的关系；
- 进一步考察行政执行网络、业务执行网络等机制通道；
- 使用固定效应、滞后解释变量、准实验分组与安慰剂检验增强识别。

#### 自动化链路

- `代码/00_master.do`：统一主控脚本；
- `代码/01_data_cleaning.do`：清洗上市公司基础信息、员工结构和违规数据；
- `代码/02_paper_analysis.do`：输出描述统计、主回归、机制检验、安慰剂检验、非线性检验、表格和图形；
- `文档/论文/mini_empirical_paper.tex`：论文主文档；
- `文档/表格/*.tex`：Stata 自动导出的表格片段；
- `文档/图片/*.png`：Stata 自动导出的论文图形。

#### 论文产出

- `stata测试/文档/论文/mini_empirical_paper.tex`
- `stata测试/文档/论文/mini_empirical_paper.pdf`

#### 说明

这是最接近“完整自动化学术论文”的样例，也是本仓库里最能体现 `Stata -> LaTeX -> 成稿` 闭环的测试。

### 2. `stata测试2`

这是一个偏理论写作的测试项目，论文主题为：

**《走人户随礼、关系资本与跨期财富配置：一个关于中国礼物流动的经济学解释》**

#### 研究内容

- 从经济学角度解释中国社会中的“走人户随礼”；
- 将礼物流动理解为嵌入关系网络中的跨期财富配置与非正式保险机制；
- 引入关系资本、事件到达率、流动性约束、霍特林式社会距离与网络效应；
- 更强调理论建模与机制推演，而不是经验回归。

#### 自动化链路特点

- 该测试更侧重 `GPT-5.4 API + LaTeX` 的理论论文生成能力；
- 通过 LaTeX 直接组织理论模型、命题、推论与图示；
- 用于验证自动化流程不一定依赖实证数据，也可以支持偏理论型论文写作。

#### 论文产出

- `stata测试2/gift_exchange_paper.tex`
- `stata测试2/gift_exchange_paper.pdf`

#### 说明

虽然目录名仍沿用 `stata测试2`，但它本质上是“理论型论文自动化”测试，用来说明同一条 AI + LaTeX 链路也能支持非实证型写作。

### 3. `stata测试3`

这是一个城市面板与营商活力的描述性实证测试，论文主题为：

**《城市企业进入退出与营商活力：基于 2000-2023 年全国市级地区-年份面板的描述性研究》**

#### 研究内容

- 使用 2000-2023 年全国市级地区-年份工商注册进入退出数据；
- 统一采用“市级地区-年份”统计层次；
- 分析企业进入率、退出率、净进入率的长期变化；
- 比较区域差异、城市排名变化与头部城市轨迹；
- 扩展到“市级地区-行业门类-年份”层次进行行业活跃度描述。

#### 自动化链路

- `code/00_master.do`：主控脚本；
- `code/01_analysis.do`：数据筛选、固定效应回归、异质性分析、城市排名与图形输出；
- `output/tables/*.tex`：自动生成的表格；
- `output/figures/*.png`：自动生成的图形；
- `paper/main.tex`：论文主文档。

#### 论文产出

- `stata测试3/paper/main.tex`
- `stata测试3/paper/main.pdf`

#### 说明

该测试更偏向“描述性城市面板 + 可视化 + LaTeX 成稿”的自动化样例，适合展示 Stata 与 LaTeX 协同生成图表论文的流程。

## 推荐复现方式

### 对于 `stata测试`

1. 检查原始数据是否齐全。
2. 在 Stata 中运行 `stata测试/代码/00_master.do`。
3. 查看 `stata测试/文档/表格` 和 `stata测试/文档/图片` 中的自动输出。
4. 编译 `stata测试/文档/论文/mini_empirical_paper.tex`。

### 对于 `stata测试2`

1. 直接编译 `stata测试2/gift_exchange_paper.tex`。
2. 查看理论模型、命题与图示结构。

### 对于 `stata测试3`

1. 准备对应工商注册进入退出数据。
2. 在 Stata 中运行 `stata测试3/code/00_master.do` 或 `stata测试3/code/01_analysis.do`。
3. 查看 `stata测试3/output/tables` 与 `stata测试3/output/figures`。
4. 编译 `stata测试3/paper/main.tex`。

## 关于数据与仓库上传

当前目录中包含多个体积很大的原始 `.dta` 文件，其中部分文件超过 GitHub 普通仓库的单文件限制。因此本仓库默认适合上传：

- Stata 脚本
- LaTeX 论文源码
- 已生成的论文 PDF
- 表格片段与图形结果
- 说明文档

而以下内容更适合本地保留或使用 Git LFS / 云盘单独管理：

- 超大原始数据文件
- 体积过大的中间数据文件

## 这个仓库验证了什么

这个项目验证的是一种面向论文写作的自动化方法：

- 可以在 `VS Code` 中统一组织研究目录；
- 可以借助 `GPT-5.4 API` 辅助论文构思、模型表达和章节写作；
- 可以通过 `Stata MCP` 把 Stata 执行能力嵌入自动化流程；
- 可以让 `LaTeX` 充当研究结果与论文成稿之间的统一媒介；
- 可以把不同风格的论文测试放进同一套工作流中进行迭代。

因此，这个仓库既是三个具体论文测试的集合，也是一个“AI + Stata + LaTeX”论文自动化原型。
